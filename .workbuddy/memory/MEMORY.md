### 学习管理模块——关键设计备忘（必读）

**数据存储结构（localStorage key命名）：**
| key | 说明 |
|-----|------|
| `roundsData_uid` | 每天累计轮数，格式：`{dayNum: 已完成轮数}` |
| `learnedDays_uid` | 已学习天数数组，格式：`[1, 2, 3, ...]` |
| `currentDay_uid` | 当前进度天数（整数） |

**碎片化累计逻辑（在每个HTML播放页面）：**
- 每轮结束 → 调用 `_saveRoundProgress(currentLoop)` 立即写 `roundsData_uid`
- 页面加载时 → 读取当天已累计轮数，显示"📌 已累计 N/7 轮"
- 累计 ≥ 7 → 写入 `learnedDays_uid`，变绿

**导出/导入进度码逻辑（在 index.html）：**
- 导出：读 `roundsData_uid` + `learnedDays_uid` 合并 → `btoa(JSON.stringify(data))` → 复制到剪贴板
- 导入：`JSON.parse(atob(code))` → 与本机数据逐天取较大值 → 同步更新 `learnedDays_uid`
- 合并原则：取较大值，永不覆盖本机已有记录

**uid参数规则：**
- URL：`?uid=xxx&name=姓名&day=N`
- uid 由姓名确定性哈希生成，同名同uid，可找回
- 无uid参数时，用默认值 `default`，正常显示
